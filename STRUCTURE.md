# 来只插件代码结构说明

## 📁 目录结构

```
astrbot_plugin_laizhi/
├── main.py                 # 主插件类 (89行)
├── core/
│   ├── __init__.py        # 模块初始化 (7行)
│   ├── database.py        # 数据库类 (215行)
│   └── handlers.py        # 命令处理器 (159行)
├── data/                  # 数据目录
│   └── laizhi_db.json     # 数据库文件
├── metadata.yaml          # 插件元数据
├── USAGE.md              # 使用说明
└── STRUCTURE.md          # 本文件
```

## 🔧 模块说明

### main.py (主插件类)
- **作用**: 插件入口，协调各个模块
- **核心类**: `MyPlugin`
- **职责**:
  - 初始化数据库和命令处理器
  - 注册命令路由
  - 插件生命周期管理

### core/database.py (数据库模块)
- **核心类**:
  - `LaizhiInfo`: 数据模型类
  - `LaizhiDB`: 数据库操作类
- **职责**:
  - 数据持久化
  - CRUD操作
  - 别名解析

### core/handlers.py (命令处理模块)
- **核心类**: `LaizhiHandlers`
- **职责**:
  - 处理所有用户命令
  - 参数验证
  - 业务逻辑处理
  - 响应生成

## 📊 代码统计

| main.py | 89 | 插件入口和路由 |
| core/database.py | 215 | 数据存储和检索 |
| core/handlers.py | 159 | 命令处理逻辑 |
| **总计** | **463** | **完整功能** |

## ✨ 重构优势

### 1. 模块化
- 每个文件职责单一
- 便于维护和测试
- 降低代码耦合度

### 2. 可扩展性
- 新增命令只需修改 `handlers.py`
- 数据库升级只需修改 `database.py`
- 不影响其他模块

### 3. 可读性
- `main.py` 只有 89 行，清晰易懂
- 业务逻辑集中在专门模块
- 代码组织更合理

### 4. 可维护性
- 问题定位更精准
- 修改影响范围更小
- 团队协作更方便

## 🚀 扩展指南

### 添加新命令
1. 在 `core/handlers.py` 中添加处理方法
2. 在 `main.py` 中注册路由
3. 更新 `USAGE.md` 文档

### 扩展数据库字段
1. 修改 `LaizhiInfo` 类
2. 更新 `to_dict` 和 `from_dict` 方法
3. 修改相关数据库操作

### 添加新功能模块
1. 在 `core/` 目录创建新文件
2. 在 `core/__init__.py` 中导出
3. 在 `main.py` 中导入使用

## 🔍 代码导航

- **查看命令实现**: `core/handlers.py`
- **修改数据库结构**: `core/database.py`
- **注册新命令**: `main.py`
- **模块导出**: `core/__init__.py`
