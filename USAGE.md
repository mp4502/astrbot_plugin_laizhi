# 来只图库插件使用说明

## 功能概述

本插件用于管理"来只"（图片收藏夹），支持创建、查询、添加图片、删除等功能，所有数据存储在JSON数据库中。

## 数据库结构

数据库文件位于：插件目录下的 `data/laizhi_db.json`

### 数据结构

```json
{
  "来只名称": {
    "name": "来只名称",
    "created_at": "2024-04-18T15:30:00",
    "image_count": 10,
    "last_used": "2024-04-18T16:00:00",
    "description": "描述信息"
  }
}
```

### 字段说明

- `name`: 来只名称（唯一标识）
- `created_at`: 创建时间（ISO格式）
- `image_count`: 图片数量
- `last_used`: 最后使用时间
- `description`: 描述信息

## 使用命令

### 1. 新建来只
```
新建<名称>
```
示例：`新建猫咪`
- 创建一个新的来只收藏夹

### 2. 查询来只
```
查询<名称>
```
示例：`查询猫咪`
- 显示指定来只的详细信息

### 3. 获取来只内容
```
来只<名称>
```
示例：`来只猫咪`
- 获取指定来只的内容，并更新最后使用时间

### 4. 添加图片
```
添加<名称>
```
示例：`添加猫咪`
- 为指定来只添加图片（计数+1）

### 5. 删除来只
```
删除<名称>
```
示例：`删除猫咪`
- 删除指定的来只及其所有数据

### 6. 查看所有来只
```
/列表
```
- 显示所有来只的列表和统计信息

## 技术实现

### 数据库类
插件内置了两个核心类：

#### LaizhiInfo
来只信息数据类，包含单个来只的所有属性。

#### LaizhiDB
数据库管理类，提供以下主要方法：
- `add_laizhi(name, description)` - 添加新来只
- `get_laizhi(name)` - 获取来只信息
- `update_laizhi(name, **kwargs)` - 更新来只信息
- `delete_laizhi(name)` - 删除来只
- `list_all_laizhi()` - 获取所有来只列表
- `get_statistics()` - 获取统计信息

**注意**: 这些类已经集成在 `main.py` 中，无需额外导入模块。

## 扩展建议

1. **图片存储**: 可扩展为实际存储图片路径或URL
2. **标签系统**: 可为来只添加标签分类
3. **搜索功能**: 支持按名称、标签搜索
4. **导入导出**: 支持数据的批量导入导出
5. **图片预览**: 集成实际的图片显示功能
6. **多图库支持**: 为每个来只添加多个子图库

## 注意事项

- 来只名称区分大小写
- 名称不能重复
- 数据文件会自动创建在插件的data目录下
- 所有时间都使用ISO 8601格式存储
- 数据库操作采用同步文件IO，确保兼容性
