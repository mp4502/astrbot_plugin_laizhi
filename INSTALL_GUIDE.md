# 🚀 来只插件v2.0.0 安装和使用指南

## 📥 安装步骤

### 1. 确保依赖已安装
```bash
pip install aiohttp>=3.9.0 aiofiles>=23.0.0
```

### 2. 重启AstrBot
重启AstrBot以加载新版本的插件

### 3. 验证安装
```
/列表
```
应该显示插件正常运行

## 🎯 快速开始

### 基础使用流程

#### 1. 创建来只
```
新建猫咪
```
创建一个名为"猫咪"的图片收藏夹

#### 2. 添加图片
```
添加猫咪 https://example.com/cute_cat.jpg
```
从URL下载图片并添加到猫咪文件夹

#### 3. 随机获取
```
来只猫咪
```
随机发送一张猫咪图片到聊天

#### 4. 查看状态
```
查询猫咪
```
查看猫咪的详细信息和图片数量

## 📸 图片功能详解

### 支持的图片格式
- ✅ JPG/JPEG
- ✅ PNG
- ✅ GIF
- ✅ WebP
- ✅ BMP

### 添加图片的方式

#### 方式1: URL添加（推荐）
```
添加猫咪 https://example.com/cat.jpg
```
自动下载到本地 `data/images/猫咪/` 文件夹

#### 方式2: 批量添加
```
添加猫咪 https://example.com/cat1.jpg
添加猫咪 https://example.com/cat2.jpg
添加猫咪 https://example.com/cat3.jpg
```

### 随机图片机制
```
来只猫咪
```
- 🎲 真正随机选择
- 📸 直接发送图片
- 🔄 每次可能不同

## 🗂️ 管理功能

### 别名系统
```
# 设置别名
别名猫咪 小猫

# 使用别名
来只小猫  # 等同于"来只猫咪"

# 查看别名
别名猫咪
```

### 查询和统计
```
# 查询特定来只
查询猫咪

# 查看所有来只
/列表
```

### 删除功能
```
删除猫咪
```
⚠️ 警告：此操作会：
- 删除来只记录
- 删除所有相关图片文件
- 释放磁盘空间
- **不可恢复**

## 🔧 高级用法

### 1. 长名称简化
```
新建很长的来只名称
别名很长的来只名称 短名
来只短名  # 使用简化后的别名
```

### 2. 图片计数同步
```
查询猫咪
→ 图片数量: 10 (实际文件: 8)
```
- 第一数字：数据库计数
- 第二数字：实际文件数
- 帮助发现数据不一致

### 3. 错误处理
```
# 下载失败
添加猫咪 https://invalid-url.jpg
→ ❌ 图片下载失败: https://invalid-url.jpg

# 来只不存在
来只不存在的来只
→ ❌ 未找到来只 '不存在的来只'

# 无图片情况
来只空来只
→ 💡 提示: 该来只暂无图片，使用 '添加空来只 <图片URL>' 来添加图片
```

## 🛡️ 安全建议

### 1. 定期备份
- 备份 `data/laizhi_db.json`
- 备份 `data/images/` 文件夹

### 2. 权限管理
- 确保插件有文件读写权限
- 确保有网络访问权限（下载图片）
- 确保有图片发送权限

### 3. 存储管理
- 定期清理不需要的来只
- 监控磁盘空间使用
- 删除不需要的图片文件

## 🐛 常见问题

### Q: 图片下载失败怎么办？
A: 检查网络连接和URL有效性，支持HTTP/HTTPS链接

### Q: 如何批量导入图片？
A: 目前需要逐个添加，未来版本会支持批量导入

### Q: 图片存储在哪里？
A: 存储在插件目录的 `data/images/<来只名称>/` 文件夹

### Q: 如何迁移数据？
A: 复制整个 `data/` 文件夹到新安装的插件目录

### Q: 删除的来只能恢复吗？
A: 删除操作不可恢复，请谨慎操作
