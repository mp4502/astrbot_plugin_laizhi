# 🖼️ 来只插件图片上下文集成功能

## 🎯 功能概述

**现在来只插件已经集成了图片上下文管理器！**

- 🔍 **自动捕获**聊天中的图片
- 📋 **会话管理**按会话隔离图片
- 🆔 **会话标识**追踪图片来源
- 📸 **智能添加**优先使用上下文中的图片

## 🚀 新的使用方式

### 1. 自动图片捕获
插件现在会自动监听聊天中的图片，并按会话进行管理：

```python
# 当有人在聊天中发送图片时
[用户发送图片]

# 插件自动捕获到图片上下文
📸 图片已捕获，会话ID: "qq:group:123456"
```

### 2. 智能图片添加
添加图片时，插件会优先使用上下文中的图片：

```
[用户发送图片]
添加猫咪

✅ 为 '猫咪' 添加图片成功！
🆔 会话: qq:group:123456
📁 已保存到: data/images/猫咪/cat.jpg
当前总数: 1
📋 会话图片数: 1
```

## 🔄 工作流程

### 完整流程

```
1. [用户发送图片到聊天]
   ↓
2. [插件自动捕获图片]
   - 存储URL
   - 记录会话ID
   - 记录时间戳
   ↓
3. [用户输入: 添加猫咪]
   ↓
4. [插件处理添加命令]
   - 从图片上下文获取最近图片
   - 显示会话信息
   - 下载图片到本地
   - 保存到来只文件夹
   ↓
5. [返回结果]
   ✅ 成功信息
   - 会话标识
   - 保存位置
   - 统计信息
```

### 图片获取优先级

```
1. 图片上下文管理器 (优先)
   ↓ (如果没有)
2. 当前消息组件
   ↓ (如果没有)
3. 返回错误提示
```


```python
self.handlers = LaizhiHandlers(self.db, self.photo_db, self.image_context_manager)
```

### 3. 在处理器中使用


## 📝 配置说明

### 图片上下文管理器配置

插件使用以下默认配置：

```python
init_image_context_manager(
    isolation_mode="session",    # 会话隔离
    max_images=20,              # 每个会话最多20张图片
    ttl_seconds=0,               # 图片不过期
    max_sessions=200,            # 最多200个会话
    include_url_in_context=True  # 包含URL在上下文中
)
```
